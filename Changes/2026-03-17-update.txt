SESSION 3 — BUG FIXES (POST-DEPLOY)

BUGS FIXED
----------
src/App.tsx

  1. Step icons incorrectly showing green on initial load (steps 2, 3, 5, 6, 7)
     Root cause: stepCompletion used schema.safeParse() against form default
     values. Steps with mostly optional fields passed validation immediately,
     making their sidebar icons appear completed (green) before the user
     interacted with them.
     Fix: replaced entire stepCompletion logic with index < currentStep.
     Steps now show as completed only after the user has moved past them.

  2. Phone number displayed in header top-right
     Fix: removed <a href="tel:6477662660"> link and Phone icon.
     Removed unused Phone import from lucide-react.
     Removed unused schema imports (MilitaryServiceSchema, EmploymentSchema,
     DriverTrainingSchoolSchema, OtherEducationSchema, UnemploymentSchema)
     that were only referenced in the old stepCompletion logic.
